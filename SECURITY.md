# Security Policy

## Модель и границы

- **Токен — это пароль от вашего учёта.** Он хранится локально в
  `~/.moysklad-mcp/cabinets.json` (chmod 600), НИКОГДА не попадает в репозиторий,
  в конфиг клиента или в чат. Сервер работает там же, где ваш агент (локально).
- **Запись за двумя гейтами.** (1) Процессный guard: запись выключена по умолчанию,
  включается `MOYSKLAD_ALLOW_WRITE=1`, опц. пин к кабинету `MOYSKLAD_WRITE_CABINETS`;
  покрывает и сырые `ms_call_method`/`ms_call_raw`. (2) Per-call гейт: `write`
  требует `confirm_write=true`, `destructive` (проведение/удаление) — ещё и
  `i_understand_this_modifies_data=true`. Без подтверждений в сеть ничего не уходит.
- **Создание делает черновик** (`applicable:false`); проведение, которое двигает
  учёт, — отдельный destructive-шаг.
- **Только тестовый кабинет** для записи, пока не проверите на нём. Не направляйте
  `MOYSKLAD_ALLOW_WRITE=1` на боевые книги.

## Если токен засветился

Сразу перевыпустите его в МойСклад (Настройки → Пользователи → Токены доступа),
старый отзовите. Не вставляйте токен в чат даже частично.

## Как сообщить об уязвимости

Заведите приватный security-advisory в репозитории (Security → Report a
vulnerability) либо напишите мейнтейнеру. Пожалуйста, не публикуйте детали
публично до фикса. Мы постараемся ответить в разумный срок.

## Что НЕ делает сервер

Не отправляет телеметрию, не «звонит домой», не передаёт ваши данные третьим лицам.
Все вызовы идут напрямую в API МойСклад по вашему токену.
